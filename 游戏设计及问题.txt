Cyclooctane文档

游戏概述：	
勇者“苯”前来打倒大魔王“环辛烷”，最终吸收魔王爆出的物质而黑化成新一代魔王 
游戏特点：
道具随机，攻击方式多样，随机房间，随节奏发射子弹（？），由种子记录游戏特征，可存取档。
基本通过房间方式为撑过一段时间，房间每个角落拥有刷怪点随音乐生成怪物
(可设计其他通过房间方式，如强行炸门或特殊房间特殊任务)

主角苯：
描述：
其基本能力为喷射眼泪，大招为将核心扔出并引发爆炸，在失去核心的一段时间内能够通过旋转房间使怪物碰到地刺
维护数据及方法：
	数据成员
	{
		1.num移动速度
		2.POINT坐标数组
		3.judge核心是否扔出（是否可旋转房间）
		4.子弹链表
		5.num血量
		6.judge能否飞行（无视障碍）
	}
	方法
	{
		1.绘制图形实现移动
		2.发射子弹
		3.旋转地图
	}

子弹
描述：由主角射出，遇墙反弹，可伤害主角
维护数据及方法：
	数据成员
	{
		1.num射程
		2.num速度
		3.POINT坐标数组
		4.static judge是否穿透障碍物
		5.static judge是否穿透怪物
		6.static judge是否能伤害主角
		7.judge是否存在
	}
	方法
	{
		1.图形绘制
	}
	
怪物：
描述：
无喷射眼泪能力，通过最短路算法绕过地刺、障碍追踪主角，碰到主角能使其扣血，有碰撞体积，通过房间角落的刷怪点生成。
维护数据及方法：
	数据成员
	{
		1.num移动速度
		2.num血量
		3.POINT坐标数组
	}
	方法
	{
		1.图形绘制
		2.自动寻路
	}
	
精英怪：
描述：
从怪物类继承，拥有更快的速度并无视碰撞体积

Boss：
描述：
从怪物类继承，每层的守关者，可以旋转地图（？）

房间：
描述：
分为三角形宝箱房，四至七边形怪物房，八边形Boss房，圆形商店
维护数据及方法：
	数据成员
	{
		1.边数
		2.坐标数组
		3.开门
	}
	方法
	{
		1.房间生成
	}

障碍
描述：无
维护数据及方法：
	数据成员
	{
		1.坐标数组
	}
	方法：
	{
		1.图形绘制
	}
地刺：继承障碍类
描述：能够扎伤主角和怪物，无法阻挡人，无法摧毁
维护数据及方法：
	数据成员
	{
		1.num伤害
		2.judge能否造成伤害（伸缩地刺）
	}
	方法：
	{
		1.定时伸缩（伸缩地刺）
	}
岩石：继承障碍类
描述：无法造成伤害，能够阻挡人，能够被摧毁
维护数据及方法：
	数据成员
	{
		1.judge是否存在（是否被炸毁）
	}


	
需要解决问题：
	1. 房间绘制及生成   难点：障碍物的分布
		{
			一号方案：设计算法随机生成 ( 需要考虑种子问题 )
			二号方案：设计几个固定分布的房间 随机选择
		}
	2. 怪物的自动寻路算法 (最短路)
	3. 整体逻辑的实现
	4. 种子算法 (哈希函数)
	5. 存取档  (文件读写)
	6. 碰撞问题
		{
			主角与怪物
			主角与静止、旋转墙壁
			主角与旋转岩石
		}
	7. 几何问题
		{
			子弹遇多边形墙壁反弹
			多边形旋转绘制
		}
